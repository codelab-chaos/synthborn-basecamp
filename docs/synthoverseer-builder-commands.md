## UI-safety policy

Commands that open dialogs, toggle HUDs, or otherwise alter the player's interface state are EXCLUDED from this catalog — the Overseer dispatches these without the player expecting a popup, which interrupts gameplay flow. Explicitly excluded examples: /buildertoolslegend, /ui-gallery, /commands, /help, /viewport. Re-evaluate any new entry against this rule before adding.

## Tag glossary

| Tag | Meaning |
|---|---|
| `requires-selection` | Caller must have set pos1 AND pos2 before this command works. Selection commands themselves don't carry this tag. |
| `writes-blocks` | Modifies world blocks (not just selection state). Reversible via /undo or /osu. |
| `reads-selection` | Reads the current selection bounds without modifying blocks. |
| `affects-clipboard` | Writes or reads the per-player clipboard. |
| `destructive` | Clears or overwrites significant amounts of world state; pair with caution. |
| `reversible-via-undo` | Joins Hytale's BuilderTools undo stack; /undo or /osu reverts. |
| `no-args` | Takes no arguments; dispatch with args={}. |
| `selection-op` | Manipulates the active selection itself (does not write blocks). |
| `history-op` | Pops or pushes the engine's edit history. |

## selection

### `/deselect`

Clear the current world-editor selection. Use between unrelated region ops so a stale bbox doesn't get reused by accident. Cheap and safe.

**Tags**: `selection-op`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "deselect", "args": {} }
```

### `/pos1`

Set the FIRST corner of the world-editor selection. Pair with pos2 to define a bounding box. Required before fillblocks / clearblocks / replace / copy / cut / move / hollow / stack / wall.

**Tags**: `selection-op`

| Param | Type | Required | Description |
|---|---|---|---|
| `x` | integer | yes | World X (absolute integer) |
| `y` | integer | yes | World Y (absolute integer) |
| `z` | integer | yes | World Z (absolute integer) |

**Examples:**

```json
{ "name": "pos1", "args": {"x":-240,"y":64,"z":-7} }
```

### `/pos2`

Set the SECOND corner of the world-editor selection. Use after pos1 to define the bounding box. Most other commands need both pos1 AND pos2 set.

**Tags**: `selection-op`

| Param | Type | Required | Description |
|---|---|---|---|
| `x` | integer | yes | World X |
| `y` | integer | yes | World Y |
| `z` | integer | yes | World Z |

**Examples:**

```json
{ "name": "pos2", "args": {"x":-230,"y":72,"z":3} }
```

## write

### `/clearblocks`

Set every cell of the current pos1..pos2 selection to Empty (air). Equivalent to fillblocks with the air id but safer and clearer in intent. Reversible via undo. Use to dig out an area before building, or to demolish a structure.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`, `destructive`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "clearblocks", "args": {} }
```

### `/fillblocks`

Fill every cell of the current pos1..pos2 selection with one block id. ALWAYS OVERWRITES — existing blocks are replaced. Block ids are Hytale snake_case (e.g. 'Rock_Stone', 'Wood_Trunk_Oak', 'Soil_Grass'). Reversible via undo. Typical 'build me a 10x10 floor of stone' usage.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`, `destructive`

| Param | Type | Required | Description |
|---|---|---|---|
| `block` | string | yes | Block id to fill the selection with |

**Examples:**

```json
{ "name": "fillblocks", "args": {"block":"Rock_Stone"} }
```
```json
{ "name": "fillblocks", "args": {"block":"Wood_Softwood_Planks"} }
```

### `/hollow`

Hollow out the interior of the current pos1..pos2 selection — keep the outer shell, set the interior to Empty. Turns a solid cube into a room with walls/floor/ceiling in one step. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "hollow", "args": {} }
```

### `/replace`

Replace every occurrence of one block id with another inside the current pos1..pos2 selection. Cells that don't match 'from_block' are left alone. The renovation primitive — use for 'change all the dirt walls to stone' instead of clearing and rebuilding.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`

| Param | Type | Required | Description |
|---|---|---|---|
| `from_block` | string | yes | Block id to match |
| `to_block` | string | yes | Block id to substitute |

**Examples:**

```json
{ "name": "replace", "args": {"from_block":"Soil_Dirt","to_block":"Rock_Stone"} }
```
```json
{ "name": "replace", "args": {"from_block":"Wood_Trunk_Oak","to_block":"Wood_Softwood_Planks"} }
```

## clipboard

### `/copy`

Copy the contents of the current pos1..pos2 selection into the player's clipboard. Source world is NOT modified. Paste later with the paste command to duplicate the region.

**Tags**: `requires-selection`, `affects-clipboard`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "copy", "args": {} }
```

### `/cut`

Like copy, but ALSO clears the source region to Empty. Useful for relocating a structure in one step: cut → move player → paste. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `affects-clipboard`, `reversible-via-undo`, `destructive`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "cut", "args": {} }
```

### `/paste`

Paste the clipboard contents into the world. Without args, anchors at the player's current pos1 (or current standing position depending on Hytale's mode). Reversible via undo.

**Tags**: `writes-blocks`, `affects-clipboard`, `reversible-via-undo`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "paste", "args": {} }
```

## transform

### `/flip`

Mirror the contents of the current pos1..pos2 selection along an axis (x, y, or z). Useful for symmetry — flip a built house to its mirror image. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`

| Param | Type | Required | Description |
|---|---|---|---|
| `axis` | string | yes | Axis to mirror across: 'x', 'y', or 'z' |

**Examples:**

```json
{ "name": "flip", "args": {"axis":"x"} }
```
```json
{ "name": "flip", "args": {"axis":"z"} }
```

### `/move`

Translate the contents of the current pos1..pos2 selection by (dx, dy, dz) blocks. The source cells become Empty; destination cells receive the moved contents. The 'shift this whole house 2 east' tool. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`

| Param | Type | Required | Description |
|---|---|---|---|
| `dx` | integer | yes | Translation along +X |
| `dy` | integer | yes | Translation along +Y |
| `dz` | integer | yes | Translation along +Z |

**Examples:**

```json
{ "name": "move", "args": {"dx":2,"dy":0,"dz":0} }
```
```json
{ "name": "move", "args": {"dx":0,"dy":4,"dz":0} }
```

### `/rotate`

Rotate the contents of the current pos1..pos2 selection by 90 / 180 / 270 degrees around the vertical (Y) axis. Useful for re-orienting a structure to face a different direction. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`

| Param | Type | Required | Description |
|---|---|---|---|
| `degrees` | integer | yes | Rotation angle: 90, 180, or 270 |

**Examples:**

```json
{ "name": "rotate", "args": {"degrees":90} }
```
```json
{ "name": "rotate", "args": {"degrees":180} }
```

## history

### `/redo`

Re-apply the most recently undone BuilderTools edit. Pair with undo for 'wait, actually keep that' flows.

**Tags**: `history-op`, `no-args`

No arguments — pass `args: {}`.

**Examples:**


### `/undo`

Undo the most recent BuilderTools edit (LIFO). Covers fillblocks / clearblocks / replace / copy / cut / paste / move / flip / rotate / hollow / stack / wall — every write command in this catalog. The engine's stack survives server restarts. Prefer /osu (Overseer slash command) when iterating mid-session — same effect, no LLM round-trip.

**Tags**: `history-op`, `no-args`

No arguments — pass `args: {}`.

**Examples:**

```json
{ "name": "undo", "args": {} }
```

## shape

### `/stack`

Stack N copies of the current pos1..pos2 selection along a direction. Useful for repeating a column / pillar / wall section. Direction is the axis with sign. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`

| Param | Type | Required | Description |
|---|---|---|---|
| `count` | integer | yes | Number of stacked copies (1-32) |
| `direction` | string | yes | Stack direction: '+x', '-x', '+y', '-y', '+z', '-z' |

**Examples:**

```json
{ "name": "stack", "args": {"count":4,"direction":"+y"} }
```
```json
{ "name": "stack", "args": {"count":3,"direction":"+x"} }
```

### `/wall`

Build a wall along the PERIMETER of the current pos1..pos2 selection out of the given block id. Sets the outer ring of cells to the block and leaves the interior alone. Useful for room walls / fences / boundaries. Reversible via undo.

**Tags**: `requires-selection`, `writes-blocks`, `reversible-via-undo`

| Param | Type | Required | Description |
|---|---|---|---|
| `block` | string | yes | Block id for the wall |

**Examples:**

```json
{ "name": "wall", "args": {"block":"Rock_Stone"} }
```
```json
{ "name": "wall", "args": {"block":"Wood_Trunk_Oak"} }
```
